<div id="preamble"> <div class="sectionbody"> <div class="paragraph"> <p>Interscript can be used as a Ruby Gem library to be integrated with other Ruby applications.</p> </div> </div> </div> <div class="sect1"> <h2 id="_gemfile">Gemfile</h2> <div class="sectionbody"> <div class="paragraph"> <p>You need to make sure your Gemfile contains the following lines:</p> </div> <div class="listingblock"> <div class="content"> <pre class="highlight"><code class="language-ruby" data-lang="ruby">source "https://rubygems.org"

gem "interscript", "~>2.0"</code></pre> </div> </div> </div> </div> <div class="sect1"> <h2 id="_requiring">Requiring</h2> <div class="sectionbody"> <div class="paragraph"> <p>In your codebase, if you don’t do <code>Bundler.require</code>, you will need to add the following line:</p> </div> <div class="listingblock"> <div class="content"> <pre class="highlight"><code class="language-ruby" data-lang="ruby">require "interscript"</code></pre> </div> </div> </div> </div> <div class="sect1"> <h2 id="_listing_all_available_maps">Listing all available maps</h2> <div class="sectionbody"> <div class="paragraph"> <p>To list all available maps, one must execute the following code:</p> </div> <div class="listingblock"> <div class="content"> <pre class="highlight"><code class="language-ruby" data-lang="ruby">maps = Interscript.maps</code></pre> </div> </div> <div class="paragraph"> <p><code>maps</code> will be an array containing all Interscript maps by their name.</p> </div> </div> </div> <div class="sect1"> <h2 id="_transliterating_text">Transliterating text</h2> <div class="sectionbody"> <div class="paragraph"> <p>To transliterate test using a given map, like <code>bas-rus-Cyrl-Latn-2017-bss</code>, one must execute:</p> </div> <div class="listingblock"> <div class="content"> <pre class="highlight"><code class="language-ruby" data-lang="ruby">cache = {} input = "Хелло" output = Interscript.transliterate("bas-rus-Cyrl-Latn-2017-bss", input, cache)</code></pre> </div> </div> <div class="paragraph"> <p>You should preserve the <code>cache</code> variable for performance reasons. It is optional, you don’t need to (but should) supply it.</p> </div> <div class="sect2"> <h3 id="_using_ruby_compiler">Using Ruby compiler</h3> <div class="paragraph"> <p>If performance is of utmost performance for your application and you want to sacrifice a little bit of loading time for much better performance, you can use <code>Interscript::Compiler::Ruby</code> instead of <code>Interscript::Interpreter</code> (which is used by default).</p> </div> <div class="listingblock"> <div class="content"> <pre class="highlight"><code class="language-ruby" data-lang="ruby">require "interscript/compiler/ruby"

cache = {} input = "Хелло" output = Interscript.transliterate("bas-rus-Cyrl-Latn-2017-bss", input, cache, compiler: Interscript::Compiler::Ruby)</code></pre> </div> </div> </div> </div> </div>
